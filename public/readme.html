SportsOdds

Australian Sports Betting Odds Comparison

⚠️ Learning Project Notice
SportsOdds is a personal homelab and DevOps learning project built to demonstrate cloud-native infrastructure, CI/CD pipelines, and GitOps practices. It is not a commercial product. Odds data is for informational purposes only — always verify with the bookmaker before placing any bet.

Overview

SportsOdds collects and displays betting odds from Australian bookmakers for AFL, NRL, Cricket, Tennis, and UFC. The app runs entirely on a self-hosted 6-node Kubernetes homelab cluster, deployed via GitOps using ArgoCD, with a full CI/CD pipeline through self-hosted GitLab.

Tech Stack

Kubernetes (kubeadm) ArgoCD GitOps GitLab CI/CD Kaniko (image builds) CloudNativePG Vault + External Secrets Cloudflare Tunnel Longhorn Storage MetalLB + ingress-nginx Prometheus + Grafana Loki + Promtail Cilium + Hubble Node.js + Express

Architecture

Component Roles

VS CodeLocal development on Windows.
GitLab (self-hosted)Code repo, CI pipeline, container registry — all in-cluster.
KanikoBuilds Docker images inside GitLab CI without privileged Docker-in-Docker.
ArgoCDGitOps controller — watches the manifest repo and syncs changes to the cluster automatically.
CloudNativePGPostgreSQL operator managing the odds database with automated backups to S3.
Vault + ESOHashiCorp Vault stores all secrets. External Secrets Operator syncs them into k8s Secrets automatically.
k8s CronJobFetches odds from the-odds-api.com twice daily (00:00 and 12:00 UTC) and writes to Postgres.
LonghornDistributed block storage across the cluster nodes for persistent volumes.
MetalLB + ingress-nginxLoadBalancer IP pool and ingress controller for routing traffic to services.
Cloudflare TunnelExposes services publicly without opening any ports on the home router. Home IP is never exposed.
Cilium + HubbleeBPF-based CNI for network policy and Hubble for real-time network observability across the cluster.
Prometheus + GrafanaMetrics collection and dashboards for cluster and application monitoring.
Loki + PromtailLog aggregation and querying across all pods in the cluster.

CI/CD Flow

Code pushed to GitLab → pipeline triggers → Kaniko builds and pushes image to self-hosted registry → pipeline commits the new image tag to the GitOps manifest repo → ArgoCD detects the change → rolls out the new version to the cluster automatically. No manual kubectl commands needed after the initial setup.
